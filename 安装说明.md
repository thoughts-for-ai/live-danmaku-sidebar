# ✅ Obsidian 弹幕侧边栏插件 安装说明

---

## 🚀 立即安装方法（不需要任何开发环境）

这个插件是原生纯JS插件，**不需要编译、不需要npm、不需要任何工具**，直接复制就可以运行。

### 步骤：
1.  打开你的 Obsidian 仓库文件夹
2.  进入隐藏文件夹 `.obsidian/plugins/`
3.  把整个 `live-danmaku-sidebar` 文件夹整个复制进去
4.  重启 Obsidian
5.  进入 设置 → 第三方插件 → 启用 "Live Danmaku Sidebar"

---

```
live-danmaku-sidebar/
```

## ⚙️ 使用方法
1.  左侧图标栏会出现一个**直播弹幕自定义图标**（按 Obsidian `addIcon` 的默认 `100x100` 视口坐标绘制，视觉上与原生图标一致），点击打开右侧弹幕侧边栏
2.  插件首次加载时（仅首次），会在你的仓库里自动创建这个文件夹：
    ```
    live-danmaku/
    ├── input/
    │   ├── 观众入场-中文.md
    │   ├── 手速惊叹-中文.md
    │   ├── 迷妹打call-中文.md
    │   ├── 直播间神回复-中文.md
    │   ├── chat-arrives-en.md
    │   ├── speed-progress-hype-en.md
    │   ├── hype-squad-en.md
    │   └── gremlin-chat-en.md
    ├── idle/
    │   ├── 停顿陪伴-中文.md
    │   └── idle-companion-en.md
    └── milestones/
        ├── 100-中文里程碑.md
        ├── 300-中文里程碑.md
        ├── 600-中文里程碑.md
        ├── 1000-中文里程碑.md
        ├── 100-milestone-en.md
        ├── 300-milestone-en.md
        ├── 600-milestone-en.md
        └── 1000-milestone-en.md
    ```
3.  你可以直接往这些文件夹里继续放 `.md` 文件，插件会自动读取：
    - `live-danmaku/input/`：输入时触发的通用弹幕
    - `live-danmaku/idle/`：停顿时触发的激励弹幕
    - `live-danmaku/milestones/`：里程碑弹幕（文件名需**以数字开头**，例如 `200.md` 或 `150-中文里程碑.md`）
4.  支持的内容格式非常简单：**每行一条弹幕**，也支持 Markdown 列表
5.  仅首次安装会自动生成默认模板；后续你对 `live-danmaku/` 的删除、修改、重命名都会被保留，插件重启时不会自动“补回”
6.  如需恢复缺失的默认模板，可打开命令面板执行：
    - `Live Danmaku Sidebar: 手动补全默认弹幕库（仅补缺失）`
7.  开始写作时，只有在检测到**有效字数增长**时才会触发输入弹幕；停顿约 5 秒后，才会切换为激励弹幕
8.  当“当前文档本次打开后的新增字数”达到里程碑阈值（如 100/300/600/1000）时，会触发更显眼的里程碑弹幕
9.  侧边栏可以随时打开/关闭，不影响其他功能

### 里程碑统计规则（重要）
- 统计维度是：**当前文档，本次打开后的新增字数**。
- 切换到另一个文档，会开启该文档的新会话统计。
- 关闭 Obsidian 后统计会自然清空，不会越用越大。
- 这是纯内存状态，不写入配置、不写入磁盘，几乎无额外性能负担。


## 💡 关于技术
- 使用官方 `editor-change` 事件监听输入
- 只在**有效字数增长**时触发输入弹幕，且采用累计阈值触发，避免“每敲一个字符就弹”
- 停顿弹幕使用低频定时检查，不做高频空转监听
- 里程碑统计采用“当前文档会话内内存计数”，无需持久化存储
- 全部 CSS 动画使用硬件加速
- 正确的资源清理机制，不会内存泄漏

你现在就可以直接把这个文件夹放到你的Obsidian插件目录运行。

## 🛠 若提示“插件加载失败”，请按这个顺序检查

1. **文件夹名称必须一致**  
   插件目录必须是：
   ```
   .obsidian/plugins/live-danmaku-sidebar/
   ```
   不能改成别的名字，因为它必须和 `manifest.json` 里的 `id` 保持一致。

2. **确认三个运行文件都在同一层级**
   ```
   live-danmaku-sidebar/
   ├── manifest.json
   ├── main.js
   └── styles.css
   ```
   不要多套一层文件夹。

3. **覆盖更新后重启 Obsidian**  
   Obsidian 有时会缓存旧插件代码，只替换文件但不重启，可能继续报加载失败。

4. **确认 Obsidian 版本至少为 1.0+**  
   本插件 `minAppVersion` 为 `1.0.0`。

5. **弹幕没显示但插件已启用**  
   请确认右侧侧边栏已经打开，并检查仓库中是否存在：
   ```
   live-danmaku/input/
   live-danmaku/idle/
   live-danmaku/milestones/
   ```
   且其中的 `.md` 文件内确实有可读取的文本内容。

   如果你想恢复预装的目录或模板文件，可在命令面板执行：
   `Live Danmaku Sidebar: 手动补全默认弹幕库（仅补缺失）`
